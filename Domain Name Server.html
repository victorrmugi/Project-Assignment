Domain Name Server

The Domain Name System (DNS) is the Internet's hierarchical, distributed naming system that translates human-readable domain names (e.g., www.google.com) into numerical IP addresses (e.g., 142.250.190.46) that computers use to identify each other on the network. It functions as the "phonebook" of the Internet.

When you type a URL into a browser, a DNS resolver contacts a series of DNS servers — starting from root servers, then top-level domain (TLD) servers, and finally authoritative nameservers — to resolve the domain to its correct IP address. This process, called DNS resolution, typically takes milliseconds and results are cached locally to improve speed.

DNS servers use a distributed database to store and manage the mappings of domain names to IP addresses, allowing for efficient and scalable resolution of domain names across the internet.

Key features of DNS include:

  • -Created by Tim Berners-Lee in 1989; publicly launched in 1991
  • -Built on three core technologies: HTML, HTTP, and URLs
  • -The Web is a service on the Internet, not the Internet itself
  • -Uses a client-server model where browsers request pages from servers
